* Client (고객, 의뢰인)
- 서비스를 요청하는 컴퓨터 HW, SW 프로그램
- DBMS에게 SQL(요청)을 보내는 모든 SW프로그램
- 자바 App, Python App

* Server (서비스 제공자)
- 서비스를 제공하는 컴퓨터 HW, SW 프로그램
- SQL(요청)을 처리하여 결과(Result)를 돌려주는 SW 프로그램(DBMS)
- IBM DB2, MS SQL-Server -> 유료
- MySQL, MariaDB, PostgreSQL, SQLite -> 오픈소스
- Redis, MongoDB, Neo4j -> NoSQL

- 시스템 고장 발생 시 얼마나 빠른 시간 내에 치료가 되어 다시 정상적으로 서비스할 수 있는 상태인지 분석하는 척도
- 백업, 장애극복 처리, 데이터 저장, 엑세스에 집중해야 함


* 다단 구조
- 클라이언트 → WEB 서버 → WAS 서버 → DBMS
- 웹 클라이언트 프로그램 : 웹 브라우저(크롬, 인터넷 익스플로러, MS 엣지 등)
- 웹 페이지 : 웹 브라우저와 web 서버 연결고리
- WEB 서버 : 클라이언트가 요청하는 정적 컨텐츠를 제공. 동적 컨텐츠를 요청받으면 WAS로 넘겨줌 (아파치, NGINX, IIS, WebtoB)
- WAS 서버 : 동적 컨텐츠를 제공하기 위한 미들웨어(서버), WEB서버가 처리하는 정적 컨텐츠도 제공 가능 (톰캣, jboss, jeus)
- 정적 컨텐츠 : HTML, CSS, image, javascript
- 동적 컨텐츠 : 위치, 날씨, 디바이스 등 사용자 정보기반 맞춤 페이지

* Transaction (거래, 처리)
- 데이터베이스 관리 시스팀의 상호작용 단위. 데이터베이스의 상태를 변화시키기 위해 수행하는 작업의 단위
- SELECT, INSERT, DELETE, UPDATE

* Integrity (무결성)
- 데이터의 정확성, 일관성, 유효성이 유지되는 것
- DBMS의 중요한 기능
- 데이터 연산에 제한을 두어 데이터의 무결성을 유지
- 개체 무결성 : 모든 테이블이 기본 키로 선택된 필드를 가져야 함, 빈 값은 허용하지 않는다
- 참조 무결성 : 두 테이블의 데이터가 항상 일관된 값을 갖도록 유지되는 것
- 도메인 무결성 : 
- 무결성 규칙

* sql : 데이터베이스에 저장된 데이터를 처리, 데이터베이스 객체를 정의, 권한을 제어하는 언어
- 관계형 DBMS에 접근하는 유일한 언어
- ANSI-SQL : 미국산업표준화기구 → 모든 데이터베이스 언어는 sql
- English-Like : sql 명령어는 대-소문자를 구분하지 않음, 문법적 구조와 의미는 영어 맞춤
- 비절차적 언어


* sql 명령어 분류
- DDL : 데이터 정의, DBA(데이터베이스관리자)만 다룸 (create, alter, drop, truncate) - 객체가 주어
- DCL : 데이터 제어 (commit(실행), rollback(취소), savepoint)
- Query : 질의어, 데이터 선택, 조회 select
- DML : 데이터 조작 (insert(삽입), update(경신), delete(삭제), merge(insert+update)) - 데이터가 주어

* 런타임 에러 : 실행 시 나오는 에러, exception 처리 가능
컴파일 에러 : 작성 시 나오는 에러


* PL/SQL : SQL + PL(Procedural Language 절차적 언어), 비절차적 언어인 절차지향으로 확장시킴
* Hadoop on SQL + Map Reduce : 하둡이라는 빅데이터 저장소에 들어간 데이터를 분석, 맵 리듀스 기법으로 자바 프로그래밍을 함
